# Agents and human operators MUST follow the rules below

This repository provides Rust bindings for the **ZWO ASI camera / EFW filter
wheel / EAF focuser SDK**:

- **`libzwo-sys`** â€” raw, unsafe FFI bindings generated by `bindgen` at build
  time from the vendored MIT SDK headers in `libzwo-sys/sdk/include/`.
- **`zwo-rs`** â€” the safe, ergonomic wrapper over `libzwo-sys`.

It is a sibling to [`qhyccd-rs`](https://github.com/ivonnyssen/qhyccd-rs) and is
consumed by the rusty-photon `zwo-camera` ASCOM Alpaca driver (see that repo's
`docs/plans/zwo-driver.md`). Scope order: **Camera â†’ EFW filter wheel â†’ EAF
focuser**.

1. **Never commit to `main`.** All work happens on a branch
   (`feature/<name>`, `chore/<name>`). Verify with `git branch` before editing.

2. **Before committing**, the `.cargo-husky` pre-commit hook runs â€” and must
   pass â€” both of:

   ```sh
   cargo clippy --all --all-targets --all-features -- -D warnings
   cargo fmt --all -- --check
   ```

   Run them yourself first. The hook installs on the first `cargo build`/`test`.

3. **Build requirements.**
   - **libclang** is required for `bindgen` (so `check`/`clippy`/`doc`/`build`
     all need it). If bindgen can't find it, set
     `LIBCLANG_PATH=/usr/lib/<triple>` (e.g. `/usr/lib/aarch64-linux-gnu`).
   - **The ZWO ASI SDK** (`libASICamera2`, `libEFWFilter`) + **libusb-1.0** must
     be on the link path to *link* â€” i.e. `cargo build`/`cargo test`. `cargo
     check`/`clippy`/`doc` do **not** link, so they only need libclang.
     `ci/install-zwo-sdk.sh` installs the SDK from the INDI mirror.
   - Override the SDK lib search dir with `ZWO_SDK_LIB_DIR=/path/to/lib`.

4. **FFI bindings are generated, not hand-written.** Do not edit generated
   bindings. To change the surface, update the vendored header in
   `libzwo-sys/sdk/include/` (keep ZWO's MIT `license.txt` alongside) and let
   `build.rs` regenerate. The headers are parsed as **C++** (`-x c++`) because
   `EFW_filter.h`/`EAF_focuser.h` use bare `bool` without `<stdbool.h>`.

5. **`unsafe` discipline.** Raw FFI lives in `libzwo-sys`. `zwo-rs` is the safe
   layer: it may use `unsafe` to call the FFI, but it must expose a sound, safe
   public API and document invariants.

6. **`simulation` feature.** A hardware-free, in-Rust environment for dev/tests.
   Mirroring qhyccd-rs, it removes the *camera*, not the SDK *link* (the SDK is
   still linked when `simulation` is on).

7. **Commits** use the author format
   `Igor von Nyssen (<AI agent name>) <igor@vonnyssen.com>`
   â€” e.g. `git commit --author="Igor von Nyssen (Claude Code) <igor@vonnyssen.com>"`.

8. **Prefer small, clear tests** that fail with a useful message (`result.unwrap()`
   over `assert!(result.is_ok())`).

9. **Persist design decisions in the repo** (README / this file / code comments),
   not in agent memory, so all operators share the same context.
